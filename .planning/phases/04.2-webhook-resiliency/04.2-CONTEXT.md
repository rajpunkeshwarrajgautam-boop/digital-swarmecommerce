- Add robust error handling for DB updates.

</domain>

<decisions>
## Implementation Decisions

### Tracking Strategy
- **Choice**: Dedicated `payment_logs` table in Supabase.
- **Rationale**: Keeps the primary `orders` table clean while providing a full audit trail.

### Idempotency Key
- **Key**: `cashfree_payment_id` (not just order ID).
- **Action**: Check if this payment ID already has a "processed" status before triggering fulfillment.

### Logging Level
- **Decision**: Log full raw payload for failed and success events to assist in debugging.
- **Storage**: `JSONB` column in Supabase.

### the agent's Discretion
- Use `supabaseAdmin` for bypass RLS and perform atomic updates.
- Standard industrial console logging for immediate server visibility.

</decisions>
