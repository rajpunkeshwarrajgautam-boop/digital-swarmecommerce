## 🏗️ The GSD (Get Shit Done) Framework

The core engine of this project is the **GSD Protocol**. It ensures that every line of code is intentional, production-ready, and architecturally sound.

### 🛠️ Core Commands
*   **/gsd:map-codebase**: Use this when you want an agent to scan the entire repo and understand the dependencies, architecture, and hidden patterns.
*   **/gsd:new-project**: Initializes a new feature or sub-system with full documentation (`PROJECT.md`, `REQUIREMENTS.md`, `ROADMAP.md`).
*   **/gsd:discuss-phase**: A collaborative session where we refine the design and logic before writing a single line of code.
*   **/gsd:execute-task**: The execution engine that performs atomic changes and verifies them immediately.

## 🚀 How to Summon an Agent

To use an agent, simply follow the **Invocation Protocol**:

1.  **The Ask**: State your objective clearly. (e.g., *"@SEO Specialist, audit our product detail pages for keyword density."*)
2.  **The Activation**: I will automatically load the corresponding `SKILL.md` (e.g., `agency-marketing-seo-specialist/SKILL.md`).
3.  **The Execution**: The agent will perform the task, provide a report, and update the project state.

## 🛡️ Operational Guidelines

*   **Zero Placeholders**: Agents never use "Lorem Ipsum" or fake data.
*   **Atomic Commits**: Every change is committed with a clear `feat:`, `fix:`, or `style:` prefix.
*   **Verification First**: No task is complete until it has been verified via browser testing or unit tests.
*   **Memory Sync**: After every session, we update `.agents/PROJECT_MEMORY.md` to ensure the next agent has full context.
